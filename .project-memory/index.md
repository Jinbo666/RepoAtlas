# Project Memory Index

## Purpose

This file is the navigation index for `.project-memory/`.

It should help an agent quickly find:
- current working state
- recent activity
- active tasks
- recent lessons
- session notes
- other dynamic memory files

Keep it short.
Do not list every trivial file.

---

## Core Entry Files

- `current-focus.md`  
  Current priorities, risks, must-remember items, and likely next actions.

- `tasks/active.md`  
  Current active tasks and their status.

- `recent-lessons.md`  
  Hot-layer lessons, pitfalls, and debugging conclusions that still matter now.

- `log.md`  
  Chronological record of important repository-memory updates.

- `source-roots.md`  
  Repository root classification: code, raw docs, curated docs, dynamic memory.

- `source-registry.md`  
  Optional registry of raw/legacy documents and their processing status.

---

## Task Files

- `tasks/active.md`
- `tasks/backlog.md`
- `tasks/done.md`

Use these for task state, not long reasoning or large design writeups.

---

## Session Notes

Directory:
- `sessions/`

Use session notes only for medium/heavy work with future value.
Do not create notes for every trivial edit.

---

## Scratch / Temporary Notes

Directory:
- `scratch/`

This area is temporary.
Do not treat scratch notes as durable truth.

---

## Navigation Guidance

For routine work, the usual entry order is:

1. `current-focus.md`
2. `tasks/active.md`
3. `recent-lessons.md` if relevant
4. recent `log.md` entries if recency matters

Then move to curated docs or code as needed.

---

## Maintenance Rule

Keep this file current when:
- major memory files are added
- old memory files are removed
- navigation structure changes materially

Do not over-maintain this file for tiny changes.
